(Refuerzo)HTML y CSS: Fundamentos del Desarrollo Web

HTML Básico

HTML (HyperText Markup Language) es el esqueleto de toda página web. Define la estructura y el contenido de tu sitio.

Estructura básica de un documento HTML:

    <!DOCTYPE html>
    <html lang="es">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>Mi Primera Página</title>
    </head>
    <body>
        <h1>Bienvenido a mi sitio web</h1>
        <p>Este es un párrafo.</p>
    </body>
    </html>

Esta estructura proporciona un documento HTML válido con los elementos esenciales.

Elementos HTML Comunes

  • <h1> a <h6>: Encabezados de diferentes niveles
  • <p>: Párrafos
  • <a>: Enlaces
  • <img>: Imágenes
  • <ul>, <ol>, <li>: Listas no ordenadas, ordenadas y elementos de lista
  • <div>: Contenedor genérico para agrupar elementos
  • <span>: Contenedor en línea para texto

Ejemplo de uso:

    <h1>Mi Blog ezequiel</h1>
    <p>Bienvenidos a mi <span style="color: blue;">blog</span>.</p>
    <img src="mi-foto.jpg" alt="Mi foto de perfil">
    <a href="https://ezequielcampos.com">Visita mi sitio web</a>
    <ul>
        <li>Primer elemento</li>
        <li>Segundo elemento</li>
    </ul>

CSS Básico

CSS (Cascading Style Sheets) se utiliza para estilizar y dar formato a las páginas HTML.

Formas de incluir CSS:

  1. En línea: Usando el atributo style en elementos HTML
  2. Interno: Usando la etiqueta <style> en el <head> del documento
  3. Externo: Enlazando un archivo .css separado

Ejemplo de CSS en línea:

    <p style="color: blue; font-size: 16px; margin-bottom: 10px;">
        Este es un párrafo con estilo en línea.
    </p>

Propiedades CSS comunes:

  • color: Color del texto
  • background-color: Color de fondo
  • font-family: Tipo de fuente
  • font-size: Tamaño de la fuente
  • margin: Espacio exterior
  • padding: Espacio interior
  • border: Borde del elemento

Git y GitHub: Control de Versiones y Colaboración

Introducción a Git

Git es un sistema de control de versiones distribuido que permite a los desarrolladores rastrear cambios en el código y colaborar eficientemente.

Características principales de Git:

  • Control de versiones distribuido
  • Branching y merging eficientes
  • Trabajo offline
  • Integridad de datos
  • Staging area para preparar commits

Git fue creado por Linus Torvalds en 2005 para el desarrollo del kernel de Linux, pero ahora es ampliamente utilizado en la industria del software.

Conceptos Básicos de Git

  • Repositorio: Contiene todo el historial y los archivos de tu proyecto
  • Commit: Una instantánea de los cambios en tu repositorio
  • Branch: Una línea independiente de desarrollo
  • Merge: Combina cambios de diferentes ramas
  • Pull: Obtiene y fusiona cambios del repositorio remoto
  • Push: Envía tus cambios locales al repositorio remoto

Comandos Git básicos:

    git init           # Inicializa un nuevo repositorio
    git add <archivo>  # Añade archivos al staging area
    git commit -m "Mensaje"  # Crea un nuevo commit con los cambios en staging
    git status         # Muestra el estado actual del repositorio
    git log            # Muestra el historial de commits
    git branch         # Lista, crea o elimina ramas
    git merge          # Fusiona ramas
    git pull           # Obtiene y fusiona cambios del repositorio remoto
    git push           # Envía commits locales al repositorio remoto

GitHub: Colaboración y Código Abierto

GitHub es una plataforma de desarrollo colaborativo basada en Git que permite a los desarrolladores alojar y revisar código, gestionar proyectos y construir software junto con millones de otros desarrolladores.

Características principales de GitHub:

  • Repositorios públicos y privados
  • Issues para seguimiento de tareas y errores
  • Pull Requests para revisión de código
  • GitHub Actions para CI/CD
  • GitHub Pages para hosting de sitios estáticos
  • Colaboración con fork y pull request

Flujo de trabajo básico con GitHub:

  1. Fork un repositorio (crea una copia en tu cuenta)
  2. Clona el repositorio a tu máquina local
  3. Crea una nueva rama para tus cambios
  4. Realiza cambios y haz commits
  5. Push los cambios a tu fork en GitHub
  6. Crea un Pull Request para proponer tus cambios al repositorio original
